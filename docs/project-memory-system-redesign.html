FrogClaw 项目级记忆系统重设计

移除“命名空间”概念，以项目作为记忆边界。系统按项目自动捕获、摘要、向量化、去重合并和召回，用户只需要确认项目与策略，不再手动维护记忆桶。

目标：项目即记忆容器 自动捕获默认开启 向量模型仍必需
设计原则 系统架构 页面原型 自动记忆流水线 数据模型 迁移方案 设置策略 实现拆分

设计原则

把 MemOS 的自动捕获思路压缩成适合 FrogClaw 的项目级方案。

项目是唯一边界

每个项目拥有一份独立记忆配置、向量索引、自动捕获队列和召回策略。删除“命名空间”入口，聊天页只选择项目。

自动写入，用户可审阅

会话结束后自动抽取候选记忆。默认先进入“待确认”，用户可切换成可信项目自动入库。

聊天模型和向量模型分工

GPT-5.x 用于摘要、抽取、过滤、合并判断；embedding 模型用于检索相似度，不能用普通聊天模型替代。

系统架构

从“手动 namespace + item”改成“project profile + memory entries + capture jobs”。

1

项目绑定

对话、文件、知识库和记忆都挂到 projectId。

2

自动捕获

assistant 完成后提取最新 user/assistant/tool 片段。

3

摘要抽取

用聊天模型输出结构化候选记忆，过滤无价值内容。

4

向量索引

用项目配置的 embedding 模型写入 mem_projectId 集合。

5

召回注入

下一轮按项目检索，经过相关性过滤后注入上下文。

自动记忆流水线

参考 MemOS，但先实现轻量可靠版本。

阶段输入处理输出
Capture 最新 user / assistant / tool 消息 清理 memory 注入、系统提示、thinking、空消息和重复消息 memory_capture_jobs
Extract 清理后的 turn 调用 gpt-5.x 输出 JSON：事实、偏好、决策、路径、错误签名、无需记忆原因 pending_memory_candidates
Dedup / Merge 候选记忆 + 相似记忆 content hash 精确去重；向量 TopK 后用 LLM 判断 new / duplicate / update 新建或更新 project_memory_items
Embed 最终摘要和内容 使用项目 embedding 模型生成向量，写入 mem_projectId ready / failed / skipped
Recall 下一轮 user prompt 项目内检索 + 全局偏好可选检索 + LLM 相关性过滤 注入上下文和召回日志
写入策略建议： 默认“先审阅后入库”。项目可切换为“自动入库”，但必须保留捕获日志、撤销、禁用某条记忆和批量清理能力。

迁移方案

从现有 memory_namespaces 平滑过渡。

  1. 新增 projects 和项目级记忆表，不立即删除旧表。
  2. 为每个已有 project 创建默认 project_memory_profiles。
  3. 旧 namespace 的 scope=project 迁移到对应 project；无法判断项目的先放到“未归档项目”。
  4. 旧 namespace 的 scope=global 迁移到“全局偏好”，作为可选跨项目召回源。
  5. 向量集合从 mem_namespaceId 重建为 mem_projectId。
  6. 前端移除 namespace 管理入口，保留只读迁移报告。
兼容风险： 当前 conversation 存的是 enabled_memory_namespace_ids。需要迁移成 memory_project_id、memory_enabled、memory_write_policy，并在读取旧会话时做兼容映射。

设置策略

用户只配置“项目如何记忆”，不再创建命名空间。

项目设置页

  • 启用项目记忆
  • 写入模式：关闭 / 待确认 / 自动入库
  • 摘要模型：默认使用当前项目聊天模型，可指定 gpt-5.x
  • 向量模型：必须选择 embedding 模型
  • 召回范围：仅项目 / 项目 + 全局偏好

对话页

  • 显示当前项目记忆状态，而不是 namespace 选择器
  • 上下文条显示“项目记忆：已召回 N 条”
  • 会话设置只保留启用/禁用本项目记忆
  • 结束后生成捕获日志，可跳到记忆页查看候选

实现拆分

建议分三期，先把核心路径跑通。

阶段范围验收标准
Phase 1 项目级 profile、UI 去 namespace、手动记忆按项目入库、项目内向量检索 用户创建/选择项目后即可添加和检索项目记忆
Phase 2 assistant 完成后自动抽取候选，待确认队列，确认后索引 完成一次对话后自动出现候选记忆，确认后下一轮可召回
Phase 3 去重合并、召回 LLM filter、捕获日志、迁移旧 namespace 重复事实不会无限增长，日志可解释每次写入和召回
